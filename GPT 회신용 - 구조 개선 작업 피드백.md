# GPT 회신용 — 구조 개선 작업 피드백

검토 의견과 작업 지시서를 반영해 크랙 작품개발 프로젝트의 1차 운영 구조 정비를 완료했다. 핵심 권고였던 **단일 정본 확정, 활성 복사본 제거, 작품 SSOT 템플릿 생성, AGENTS.md 강화**를 모두 적용했다.

이 문서는 적용 결과와 보류 항목을 전달하고, 첫 작품 파일럿 개발 착수 가능 여부를 다시 검토받기 위한 회신이다.

---

## 1. 채택한 핵심 결정

### 단일 정본

권장안 A를 채택했다.

```text
C:\Users\kmins\OneDrive\문서\크랙
```

이 폴더를 Codex 작업공간이자 Obsidian 볼트로 직접 사용한다. 프로젝트 루트에 `.obsidian/app.json`을 생성했고, Obsidian에서 현재 `README - 크랙 - Obsidian` 창이 열리는 것을 확인했다.

결정 근거는 다음 ADR에 기록했다.

`05_결정_기록/ADR_0001_정본위치_결정.md`

### 기존 Obsidian 복사본

기존 활성 복사본은 더 이상 기존 볼트에 남아 있지 않다.

기존 위치:

```text
C:\Users\kmins\Documents\Obsidian Vault\뤼튼 캐릭터 제작 지식베이스
```

삭제하지 않고 다음 위치로 이동해 비활성 백업으로 보존했다.

```text
90_아카이브/legacy_obsidian_vault_copy/
```

기존 볼트 루트에 만들었던 진입 노트들도 다음 위치로 이동했다.

```text
90_아카이브/legacy_obsidian_vault_entrypoints/
```

따라서 활성 작업 위치에는 정본이 하나뿐이며, 아카이브 자료는 AGENTS.md에 따라 정본 근거로 사용하지 않는다.

## 2. 기존 자료 이동 및 보존

기존 지식베이스를 다음 위치로 이동했다.

```text
기존: 뤼튼 캐릭터 제작 지식베이스/
현재: 01_지식베이스/뤼튼 캐릭터 제작/
```

원문과 기존 활용 노트는 삭제하거나 덮어쓰지 않았다. `01 올인원 가이드.md` 183KB 보존본도 그대로 유지했다.

기존 중첩 `.obsidian` 설정은 삭제하지 않고 다음 위치로 이동했다.

```text
90_아카이브/old_nested_obsidian_settings/
```

기존 AGENTS.md 역시 다음 위치에 백업했다.

```text
90_아카이브/AGENTS_백업_20260620.md
```

구조 개선 전 GPT 제출 보고서는 현재 상태와 혼동되지 않도록 다음 위치로 이동했다.

```text
90_아카이브/GPT 검토 자료/구조 개선 전 구축 보고서.md
```

## 3. 생성한 최상위 운영 구조

현재 활성 구조는 다음과 같다.

```text
크랙/
├─ .obsidian/
├─ AGENTS.md
├─ README.md
├─ 00_운영/
├─ 01_지식베이스/
├─ 02_작품/
├─ 03_프롬프트_라이브러리/
├─ 04_테스트_로그/
├─ 05_결정_기록/
├─ 90_아카이브/
└─ 99_임시/
```

### 운영 문서

다음을 생성했다.

```text
00_운영/00 프로젝트 홈.md
00_운영/01 작업 원칙.md
00_운영/02 파일 명명 규칙.md
00_운영/03 메타데이터 규칙.md
00_운영/04 AI 작업 규칙.md
00_운영/05 변경 기록.md
```

루트 `README.md`에는 프로젝트 목적, 정본 위치, 폴더별 역할, 복사본 금지 원칙, 기본 작업 흐름을 기록했다.

## 4. 작품별 SSOT 템플릿

다음 템플릿을 생성했다.

```text
02_작품/_작품 템플릿/
├─ 00 작품 홈.md
├─ 01 작품 개요.md
├─ 02 핵심 재미와 타깃.md
├─ 03 세계관 정본.md
├─ 04 등장인물/캐릭터_템플릿.md
├─ 05 시스템 규칙.md
├─ 06 상태창 설계.md
├─ 07 키워드북 설계.md
├─ 08 프롬프트 정본.md
├─ 09 시작 상황과 예시대화.md
├─ 10 미디어 계획.md
├─ 11 테스트/
│  ├─ 테스트 계획.md
│  ├─ 테스트 케이스.md
│  └─ 테스트 결과 로그.md
├─ 12 결정 기록.md
├─ 13 변경 기록.md
├─ 14 공개 패키지.md
└─ assets/
   ├─ images/
   └─ exports/
```

각 문서는 빈 파일이 아니라 실제 작성 항목, 표, 체크리스트, 테스트 형식을 포함한다.

`08 프롬프트 정본.md`에는 다음을 적용했다.

- 초기 버전 `v0.1.0`
- 상태 `active`
- 마지막 테스트 필드
- 현재 활성 프롬프트의 단일 출처라는 안내
- 변경 기록과 테스트 결과 로그 연결

`테스트 결과 로그.md`에는 `TEST-0001` 형식과 프롬프트 버전, 기대·실제 결과, 문제, 수정, 판정 필드를 포함했다.

## 5. 지식베이스 보강

대형 원문은 보존하고 별도 정리 노트를 만드는 권고를 적용했다.

### 주제별 정리

```text
정리/프로필 작성법.md
정리/상세 프롬프트 구조.md
정리/예시대화 작성법.md
정리/상태창 설계.md
정리/키워드북 운용.md
정리/미디어 사용법.md
정리/비공개 테스트.md
정리/공개 전 점검.md
```

### 추가 체크리스트

```text
활용 노트/상태창 제작 체크리스트.md
활용 노트/테스트 체크리스트.md
활용 노트/공개 전 체크리스트.md
```

### 원문 보강 체계

```text
원문 보강/README.md
원문 보강/첫작품_필수자료/README.md
```

545개 접기 전체를 당장 이전하지 않고, 첫 작품에서 실제 사용할 자료만 상태창 → 프롬프트 구조 → 예시대화 → 키워드북 → 테스트 → 미디어 → 공개 점검 순서로 보강하도록 규칙을 기록했다.

### 검증 필요 문서

```text
검증 필요/뤼튼_서비스_사양_검증필요.md
검증 필요/모델_출력량_검증필요.md
검증 필요/공개정책_검증필요.md
검증 필요/이미지기능_검증필요.md
```

각 문서는 `doc_type: verify`, `needs_verification: true`, 낮은 신뢰도로 설정했다.

### 기존 문서 메타데이터

기존 홈, 원문 5개, 기존 활용 노트에도 `doc_type`, `project`, `status`, `source_type`, `confidence`, `needs_verification`, `last_reviewed`를 보강했다.

## 6. 공통 라이브러리와 테스트

다음을 생성했다.

```text
03_프롬프트_라이브러리/
├─ README.md
├─ 캐릭터_프로필_프롬프트.md
├─ 시뮬레이션_시스템_프롬프트.md
├─ 상태창_프롬프트.md
├─ 예시대화_생성_프롬프트.md
└─ 검수_프롬프트.md

04_테스트_로그/
├─ README.md
└─ 공통_테스트_케이스.md
```

도구 프롬프트는 작품 정본이 아니며, 출력 결과를 사용자 확정 전까지 초안으로 취급하도록 명시했다.

공통 테스트에는 역할 인식, 말투, 설정 방어, 상태 추적, 조건부 정보, 장기 일관성, 출력 형식 항목을 포함했다.

## 7. AGENTS.md 강화

기존 AGENTS.md를 백업하고 다음 규칙을 포함한 새 버전으로 교체했다.

- 단일 정본 위치
- 문서 충돌 시 우선순위
- 작업 전 작품 홈·SSOT·결정·변경·테스트 검색 절차
- 지식베이스의 활용 노트 → 정리 → 원문 순서
- 검증 필요 정보의 범위
- 사용자 확정 전 AI 제안을 `draft`로 취급
- 프롬프트 버전 상승과 테스트·변경 기록 연결
- 작품당 활성 프롬프트 하나
- 새 문서 메타데이터 규칙
- 아카이브를 활성 근거로 사용하지 않는 규칙
- 민감한 원문 예시의 무분별한 재사용 금지
- 작업 완료 보고 형식

새 대화에서도 참고자료보다 작품 SSOT가 우선하도록 명확히 했다.

## 9. 의도적으로 보류한 작업

### 첫 작품 폴더 생성

작품명과 핵심 콘셉트가 아직 정해지지 않아 실제 작품 폴더는 만들지 않았다. 대신 즉시 복제 가능한 완성형 템플릿을 준비했다.

### 545개 상세 접기와 전체 이미지 이전

검토 의견대로 전체 이전은 보류했다. 첫 작품에서 실제로 사용하는 자료만 `원문 보강/첫작품_필수자료`에 보존한다.

### 현재 서비스 사양 검증

구조 정비와 사실 검증을 분리했다. 모델, 출력량, 공개 정책, 이미지 기능은 검증 문서만 만들었으며 아직 최신 공식 자료로 채우지 않았다.

### 아카이브 완전 삭제

안전성을 위해 이전 복사본과 설정은 삭제하지 않았다. 활성 경로에서 분리하고 아카이브로 이동했으며 정본으로 사용하지 않도록 규칙화했다.

## 10. 재검토 요청

현재 상태를 기준으로 다음을 판정해 달라.

1. 단일 정본과 활성 복사본 제거가 충분히 해결되었는가?
2. 작품 템플릿의 SSOT 범위와 문서 분리가 첫 파일럿에 적절한가?
3. 프롬프트 버전 → 테스트 결과 → 변경 기록 연결 방식이 초기 운영에 충분한가?
4. 강화된 AGENTS.md의 문서 우선순위와 검색 절차에 빠진 규칙이 있는가?
5. 첫 작품 폴더를 생성해 `v0.1.0` 파일럿 개발에 착수해도 되는가?
6. 착수 전에 반드시 수정해야 할 치명적 문제가 남아 있는가?

### 현재 자체 판정

```text
정본 통합: 완료
작품 SSOT 템플릿: 완료
AGENTS.md 강화: 완료
프롬프트·테스트·변경 연결: 완료
지식베이스 주제별 진입점: 완료
최신 서비스 검증: 미착수
첫 작품 실자료 보강: 작품 기획 후 진행
첫 파일럿 개발 준비: 가능
```

검토 결과에서 치명적 문제가 없다면 다음 단계는 다음 순서로 진행할 예정이다.

```text
작품 콘셉트 확정
→ 템플릿 복제
→ 작품 개요·핵심 재미 작성
→ 필요한 원문만 보강
→ 프롬프트 v0.1.0 작성
→ TEST-0001부터 실행
```

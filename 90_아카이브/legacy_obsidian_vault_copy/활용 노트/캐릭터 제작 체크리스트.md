# 캐릭터 제작 체크리스트

## 1. 기획

- [ ] 사용자가 한 문장으로 이해할 핵심 콘셉트가 있는가
- [ ] 1:1 / 다인물 / 시뮬레이션 중 대화 구조를 정했는가
- [ ] 반드시 지켜야 할 규칙과 있으면 좋은 연출을 분리했는가
- [ ] 장기 기억이 필요한 정보와 1회성 시작 정보를 구분했는가

## 2. 프로필

- [ ] 썸네일 우하단에 얼굴·텍스트 같은 핵심 요소를 두지 않았는가
- [ ] 원형 아이콘과 앱·웹 화면에서 잘리는 범위를 확인했는가
- [ ] 제목과 실제 캐릭터 이름이 다르면 상세 설정에서 명시했는가
- [ ] 한 줄 소개 앞부분에 핵심 매력과 검색 키워드를 배치했는가

## 3. 상세 프롬프트

- [ ] 역할, 세계관, 성격, 말투, 행동 원칙을 섹션으로 나눴는가
- [ ] 같은 개념에 여러 이름·언어를 섞지 않았는가
- [ ] 중요한 규칙을 앞쪽에 두고 중복 표현을 줄였는가
- [ ] 예외 상황과 금지 규칙이 서로 충돌하지 않는가
- [ ] 예시대화가 규칙을 실제 출력 형태로 보여주는가

## 4. 상태·기믹

- [ ] 상태창 수치는 실제로 추적 가능한 수준으로 단순한가
- [ ] 키워드북이 켜지는 조건과 우선순위를 점검했는가
- [ ] 동시에 활성화될 항목 수가 과도하지 않은가
- [ ] 시작 프롬프트 확장이 필요하면 첫 인사 키워드를 활용했는가

## 5. 미디어

- [ ] 이미지 제목보다 설명이 핵심 맥락을 충분히 담는가
- [ ] 이미지 키워드가 중복되거나 환각을 유발할 만큼 모호하지 않은가
- [ ] 이미지가 없어도 대화가 자연스럽게 진행되는가
- [ ] 프로필·미디어가 현재 공개 정책에 맞는지 확인했는가

## 6. 비공개 테스트

- [ ] 첫 응답에서 이름·성격·말투가 맞는가
- [ ] 예상 밖의 사용자 행동에도 설정이 유지되는가
- [ ] 제3자 등장, 시간 경과, 장면 전환을 시험했는가
- [ ] OOC 디버그 질문으로 어떤 규칙이 충돌하는지 확인했는가
- [ ] 키워드북이 사용자 발화와 캐릭터 발화에서 각각 언제 켜지는지 확인했는가
- [ ] 앱과 웹 양쪽의 썸네일·아이콘·출력 형태를 확인했는가

## 7. 공개 전

- [ ] 공개/비공개 전환이 신규 공개처럼 노출되는지 현재 정책을 확인했는가
- [ ] 연령·세이프티 분류를 확정했는가
- [ ] 설명과 태그에 실제 사용자가 검색할 단어를 넣었는가
- [ ] 최종 프롬프트와 이미지 원본을 별도 백업했는가

# AI 작업 규칙

- 작업 시작 전에 현재 작품의 홈과 관련 SSOT를 확인합니다.
- 사용자 최신 지시와 정본 문서가 참고자료보다 우선합니다.
- 사용자가 확정하지 않은 제안은 정본에 기록하지 않습니다.
- 자료를 인용할 때 원문·AI 요약·검증된 사실을 구분합니다.
- 한 번의 프롬프트 수정에서 가능한 한 한 종류의 변수만 바꿉니다.
- 수정 후 테스트 케이스, 결과 로그, 변경 기록을 연결합니다.
- 모델·정책·글자 수 등 변동 정보는 최신 검증 없이는 단정하지 않습니다.
- 민감하거나 노골적인 원문 예시는 작품 목적과 사용자 요청 없이 재사용하지 않습니다.
- 민감 자료는 구조·형식·체크리스트 수준에서만 참고하며, 실제 출력 문장이나 노골적 표현을 자동 삽입하지 않습니다.
- 공개 가능성, 플랫폼 정책, 사용자 안전 기준에 영향을 주는 내용은 `검증 필요` 또는 공개 전 체크리스트에서 별도로 확인합니다.
- 테스트 로그가 없는 프롬프트를 `public-ready` 또는 `final`로 승격하지 않습니다.
- OneDrive 충돌·중복 파일은 임의로 병합하거나 정본화하지 않고 사용자에게 보고합니다.

세부 실행 규칙은 프로젝트 루트의 `AGENTS.md`가 우선합니다.
